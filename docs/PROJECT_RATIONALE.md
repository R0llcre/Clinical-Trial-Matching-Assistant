Project Rationale

背景
临床试验数量庞大, 信息分散在不同页面与条目中, 试验的入排标准通常是长段文本, 普通用户或协调员很难快速判断是否可能匹配。公开的 ClinicalTrials.gov 数据虽然可获取, 但缺少可解释的结构化规则。

项目存在的意义
- 降低寻找试验的成本
- 提供可解释的资格条款清单
- 让沟通与筛选更高效

项目目标
1. 试验检索与浏览
- 快速找到相关试验

2. 资格条款解释
- 把 eligibility 文本转换为可读清单

3. 匹配与候选筛选
- 基于患者画像返回 Top-K 试验

4. 可复现与可审计
- 每个结果可以追溯来源与版本

里程碑目标总览
M0 基础骨架
- 建立项目结构与统一启动方式
- 提供健康检查与就绪探针

M1 试验浏览
- 外部 API 拉取并入库
- 前端可搜索与查看详情

M2 画像与匹配
- 创建患者画像
- 生成 Top-K 匹配结果

M3 规则解析
- eligibility 文本结构化
- checklist 解释

M4 评估
- 评估数据集与指标
- 输出评估报告

M5 LLM 增强
- 可选 LLM 解析
- 失败可降级

边界说明
- 不提供医疗建议
- 不处理真实个人健康隐私数据
