Project Overview

**项目愿景**
构建一个可用、可解释、可审计的临床试验信息检索与资格条款解释系统，帮助用户更快定位潜在试验并理解入排标准。系统只做信息服务，不提供医疗建议或入组判断。

**项目背景与问题**
- 临床试验数量庞大且信息分散，用户难以快速定位相关试验。
- 入排标准多为长段文本，缺少结构化解释与可审计依据。
- 公开数据可获取，但缺少可复现的工程化检索与解释流程。

**项目目标**
- 试验检索与浏览: 用户可搜索并查看试验详情与原始入排标准。
- 资格条款解释: 将 eligibility 文本转为可读清单，并标注证据片段。
- 匹配与候选筛选: 基于患者画像生成 Top-K 候选试验。
- 可复现与可审计: 所有结果可追溯到来源数据与解析版本。

**范围与边界**
- 只做试验检索、筛选、浏览与解释。
- 支持手动患者画像与合成患者输入。
- 不接入真实 EHR 或可识别个人健康信息。
- 不提供医疗建议、入组判断或报名流程。

**关键用户路径**
1. 试验浏览
用户输入疾病或条件，系统返回试验列表并可查看详情与入排标准原文。

2. 画像匹配
用户创建患者画像，系统返回 Top-K 候选试验，并提供条款级解释与缺失信息提示。

3. 结果沟通
用户保存或导出匹配结果，用于与研究团队或医生沟通。

- 成本控制与降级策略

**成功标准**
功能层
- 三条核心路径可跑通: 搜索、匹配、解释。
- 解析失败不影响主流程，必须可降级。
- 任一试验结果可追溯来源与更新时间。

质量层
- Top-10 HitRate >= 0.70。
- 关键字段解析 F1 >= 0.80。
- 事实一致性错误率 <= 2%。

性能层
- 搜索接口 P95 < 1.5s。
- 匹配接口 P95 < 3s。

**主要风险与对策**
- 外部 API 限流或不稳定: 本地缓存 + 退避重试。
- 解析覆盖不足: 规则解析兜底 + UNKNOWN 标记。
- LLM 幻觉: 结构化校验 + 证据对齐 + 降级。
- 产品边界被误解: 明确免责声明与不提供医疗建议。
